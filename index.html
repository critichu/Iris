Iris

expanding the palette of microbial phenotypic readouts

View project on GitHub

What's this all about

Iris is an image processing software that automatically quantifies several features of microbial colonies in high throughput. While most available software can quantify colony size, Iris adds several more phenotypes. Also Iris can be easily expanded to new phenotypes, by writing a new readout module.

Using Iris

Iris is written in Java, which makes it compatible with most operating systems (Windows, macOS, *nix). Java software is typically distributed in JAR files (.jar ending), which you can run by a double-click (see below command-line alternatives). Next, click "open folder", and select a folder which holds the pictures to be processed. Iris will process any pictures with a ".jpg" file ending, displaying any messages on the application window. You can abort the process at any time, by closing the program.

  1. Running Iris 1.1 Iris GUI To run Iris as a window (GUI) application, open a command line window, cd into the folder the Iris JAR file is in, and run the following command:

java -Xmx1g -jar Iris_0.9.x.xx.jar

Note: the Iris jar file can also be run on all systems by double clicking. To do this, first make sure you have permanently set the maximum Java heap size to 1g (1 Gigabyte). Google on how to do that in your particular system/setup.

1.2 Iris console (e.g. for scripting) To run Iris as a console application (no GUI window), which can be rather useful for scripting, you need to provide Iris with a profile name and a folder that pictures are found in:

java -Xmx1g -jar Iris_0.9.x.xx.jar "profile name" "folder location‚Äù

Valid profile names are the following: "Stm growth", "Ecoli growth", "Ecoli opacity", "Ecoli opacity 384", "Ecoli growth 384 - hazy colonies", "B.subtilis Opacity (HSB)", "B.subtilis Sporulation (HSB)", "Biofilm formation", "Biofilm formation (HSB)", "Morphology profile [Candida 96-plates]" NOTE: please keep the quotes when inputting the profile name

  1. Iris output Iris outputs 2 kind of files and a log file.

2.1 .iris files These hold the results of the feature extraction process. They are in fact plain text tab-delimited files, which can easily be opened with any text viewer or Excel.

2.2 .grid files these are currently saved mainly for debugging purposes, as they demonstrate the results of gridding algorithm.

2.3 .log files Each time Iris is run, it creates a timestamped log file. This is a text file that is currently used for debugging reasons; any error messages will also be written on this file.
